# Business Analyst (Adam)
* **Student Finance Tracker Application**

  * **Overall UI/UX**

    * User-friendly GUI with a neat, orderly layout and easy navigation
    * Search bar with a dropdown list of features students may be looking for
    * Imagery to improve user-friendliness + a clean, consistent color palette
    * Welcome message displayed on the homepage
  * **Core Homepage Buttons/Sections**

    * Balance page
    * Scholarship/loans page
    * Points page
    * Optional: schedule/timetable section
    * Homepage offers (e.g., discounts on student-relevant items like laptops)
  * **Balance Page**

    * Button(s) linking to a **Bank Page** showing the current balance with real-time tracking
    * Bank page includes a section showing accumulated points
    * Expense categories (e.g., food, rent, books)
    * Analytics page highlighting spending trends (e.g., “You spent 20% more on food.”)
  * **Points System**

    * Points earned based on variables such as:

      * grades-based (more points for higher grades)
    * Points can be redeemed via the scholarship/discount system for items/services
    * Optional: level system that affects how many points are earned
  * **Scholarship System**

    * Eligibility based on average grades (e.g., 50 not eligible; 75 eligible)
    * Accounts for:

      * Subjects/course (to match scholarship field/school)
      * Student’s country/location
      * Institute providing the scholarship
      * Optional: extracurricular activities as an influencing factor
    * Matching system example: uses a student profile (grades, course, location, financial status) to suggest best-fit scholarships
  * **Safety System**

    * Login attempt counter
    * After 3 failed attempts: timed out for 5 minutes
    * Database storing user credentials linked to the program for authentication/security features
