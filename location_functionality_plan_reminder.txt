LOCATION FUNCTIONALITY — PLAN PROGRESS REMINDER (MakeCents)
============================================================
Set aside for now; resume when product needs location again.

PRODUCT PLAN (AGREED)
---------------------
- Country is chosen only during onboarding, not on every app launch.
- Flow: target-style button → location permission → GPS + reverse geocode → ISO country_code → Data Connect lookup → save user with country FK in one registration step.
- Database (when built): table Country (id, country_code, continent); User.country_id (FK), nullable if user skips.
- Login: load user’s country_id → resolve country_code + continent for scholarships / UI (prefer one GetUserProfile query with nested country).
- Scholarship filtering (later): global + same country + same continent; not implemented yet.

CLIENT (FLUTTER) — WHAT EXISTED AND WAS REMOVED
-----------------------------------------------
- Earlier addition: geolocator, geocoding, lib/location_provider.dart, lib/location/location_service.dart, lib/location/continent_mapper.dart, MultiProvider LocationProvider, refresh on HomeScreen init.
- All of the above was removed to match “onboarding-only + Data Connect” and to drop unused global location.
- pubspec.yaml: geolocator and geocoding removed (re-add only when implementing onboarding GPS).
- main.dart: no LocationProvider; _refreshSessionData does not refresh location.

DATA CONNECT — WHAT WAS ADDED (CONNECTOR ONLY)
----------------------------------------------
- Active connector: dataconnect/example/queries.gql (see dataconnect.yaml connectorDirs).
- New query: GetCountryByCode($countryCode: String!) @auth PUBLIC — countries where countryCode eq, limit 1, returns id, countryCode, continentCode.
- StoreUserProfile: optional variable countryId (UUID) passed to user_upsert.
- GetUserProfile: nested country { id countryCode continentCode }.
- Schema and country seeding were explicitly deferred; these operations will not compile/codegen until Country exists on User in dataconnect/schema/schema.gql (and table seeded).

WHEN RESUMING WORK
------------------
1. Add Country type + User.country relation in schema; deploy/migrate; seed countries.
2. firebase dataconnect:sdk:generate (or project’s usual) → update lib/dataconnect_generated.
3. Add geolocator + geocoding; onboarding UI (target button, permission, resolve country_code).
4. Call GetCountryByCode then StoreUserProfile.countryId at final registration step.
5. Extend UserProvider (or profile model) to persist country from GetUserProfile after login.
6. Android/iOS location permission strings in manifest / Info.plist.
7. Scholarship list filtering tied to profile country/continent when schema allows.
